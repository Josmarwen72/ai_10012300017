System Pipeline

RAG System Architecture

Our RAG (Retrieval-Augmented Generation) system processes queries through multiple stages to provide accurate, context-aware responses.

📝

Query Input

User submits query through the web interface

• Natural language processing • Query analysis • Intent detection
→
🔍

Retrieval

System retrieves relevant document chunks from vector store

• Dense vector similarity search • BM25 keyword matching • Hybrid scoring algorithm
→
📊

Reranking

Retrieved chunks are reranked by relevance

• Score normalization • Relevance filtering • Top-k selection
→
🤖

LLM Generation

Context is sent to LLM for response generation

• Prompt engineering • Context injection • Response generation
→
✅

Output

Final response is delivered to user

• Answer formatting • Source attribution • Quality scoring

Available Modes

🔄

RAG Hybrid

Combines dense vector search with BM25 keyword matching for optimal retrieval

• Semantic search • Keyword matching • Hybrid scoring
🎯

RAG Dense

Uses only dense vector similarity for semantic search

• Vector embeddings • Cosine similarity • Semantic matching
💬

LLM Only

Bypasses retrieval, uses LLM knowledge only

• No retrieval • Pure generation • Model knowledge

System Configuration

Vector Store

FAISS index with dense embeddings

Embedding Model

Sentence-transformers for document encoding

LLM Provider

Groq API with multiple model options

Chunking Strategy

Sliding windows with overlap for context preservation
